# 2026-07-23 fund watchlist after-hours fix

## User-reported issue
- 非交易时段看不到：
  - 新增自选基金
  - 以及白天没有看过估值的基金的估值

## What I inspected
- `app.py` fund watchlist loading/render flow
- `src/fund_watchlist_dashboard.py`
- `src/fund_estimate_capture.py`
- `src/fund_estimate_snapshot_store.py`
- `src/fund_nav.py`

## Root cause found
In `src/fund_watchlist_dashboard.py`, `build_fund_watchlist_item()` only exposed:
- `closing_estimate_pct`
when `estimate_date == nav_date`.

That is correct for calculating same-day deviation, but it hides a valid latest closing estimate for:
- newly added watchlist funds after 15:00
- funds that did not have a same-day cached estimate attached during the day
- any case where latest estimate snapshot date is newer than the latest published nav date

The UI already has fallback paths in `app.py` / `fund_object_page.py` that look for:
- `latest_closing_estimate_pct`
- `latest_closing_estimate_date`

But `build_fund_watchlist_item()` was not populating those fields when called with `estimate_snapshot` directly.
So after-hours cards could end up showing blank estimate values even though a latest snapshot existed.

## Patch applied
Updated `src/fund_watchlist_dashboard.py`:
- always populate `latest_closing_estimate_pct` from incoming `estimate_snapshot`
- always populate `latest_closing_estimate_date` from incoming `estimate_snapshot`
- keep `closing_estimate_pct` and deviation calculation gated by `estimate_date == nav_date`
