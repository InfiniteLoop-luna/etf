## 1. 当前系统现状总结

当前 ETF 项目里，个股查询页的“未来走势分析（V3）”并不是纯黑盒机器学习，也不是深度学习模型，而是：

> **规则因子打分 + 历史相似样本模型融合**

它会输出：
- 趋势状态（上涨趋势 / 震荡偏强 / 震荡整理 / 震荡偏弱 / 下跌趋势）
- 趋势分（0~100）
- 未来 5 日上涨概率
- 未来 20 日上涨概率
- 风险分（0~100）
- 支撑位 / 压力位

### 1.1 当前核心实现位置

主要逻辑位于：
- `app.py` 中的 `build_security_trend_analysis(...)`
- 页面展示位于：`render_security_trend_analysis(...)`

### 1.2 当前规则层使用的技术指标

现有规则层会基于日线时序计算以下特征：
- 均线：`MA5 / MA10 / MA20 / MA60`
- 区间收益：`ret_5 / ret_20`
- 动量：`RSI14`
- 趋势动能：`MACD`
- 波动与位置：`布林带位置 boll_pos`
- 风险：`20日波动率 volatility20`
- 价格区间：`20/60日支撑位、压力位`
- 量能辅助：`volume_ratio / turnover_rate`

### 1.3 当前规则层输出机制

当前规则层不是训练出来的，而是人工设定的打分与概率修正规则：
- 趋势分初始值通常从 `50` 起步，根据上面这些指标逐项加减分；
- 未来 5 日/20 日上涨概率从 `0.50` 起步，再根据均线位置、收益动量、RSI、MACD、量比、布林带位置进行加减；
- 风险分根据波动率、跌破均线、过热信号等进一步叠加。

这种方法的优点是：
- 可解释；
- 上线简单；
- 对数据要求低；
- 即使模型缺失也能继续跑。

缺点是：
- 参数依赖人工经验；
- 难以自动适应市场风格切换；
- 概率的统计意义不够强；
- 缺少严格的 out-of-sample 验证；
- 规则之间可能有重复计分或相关性冲突。

### 1.4 当前模型增强层

系统里还存在一个轻量模型接口：
- `score_security_timeseries_model(...)`

从调用方式看，它返回：
- `prob_up_5d`
- `prob_up_20d`
- `neighbor_count`
- `median_distance`
- `avg_future_ret_5d`
- `avg_future_ret_20d`

从这些字段推断，这一层更像：

> **相似样本 / KNN 风格模型**

其用途不是替代规则，而是对规则结果做“概率修正”。最终概率是：

> **规则概率 × (1 - 模型权重) + 模型概率 × 模型权重**

而模型权重会根据：
- 相似样本数（neighbor_count）
- 相似距离（median_distance）

做动态调整。

### 1.5 当前还有哪些辅助趋势数据

除个股页 V3 分析外，系统里还沉淀了这些与趋势相关的数据：

1. **周/月 EMA 技术信号**
   - `ts_stock_technical_signals`
   - 包含：`w_ema5 / w_ema30 / m_ema5 / m_ema30`
   - 以及：
     - `is_weekly_ema_bearish`
     - `is_monthly_ema_bearish`

2. **每日趋势推荐结果**
   - `trend_reco_runs`
   - `trend_reco_items`
   - 字段包括：
     - `trend_score`
     - `risk_score`
     - `prob_up_5d`
     - `prob_up_20d`
     - `industry`
     - `reco_type`
   - 说明系统已开始积累“历史推荐—后续表现”的验证样本。

3. **推荐策略效果自适应调权**
   - 现有代码里会基于历史命中率，动态调整 rule/model 权重。
   - 这是一个很好的方向，说明系统已经具备从“规则展示”往“可评估决策引擎”演进的基础。

## 3. 升级目标：从“可解释展示”升级为“可验证预测系统”

我建议把目标拆成三层：

### 3.1 第一层：预测要“可验证”
必须先明确预测目标（label），否则模型再复杂也没有意义。

建议把目标拆开，不要只做一个模糊的“未来走势”。

可定义的目标包括：

1. **方向类目标**
   - `y_up_5d`：未来 5 个交易日收益是否 > 0
   - `y_up_10d`
   - `y_up_20d`

2. **收益类目标**
   - `ret_5d`
   - `ret_10d`
   - `ret_20d`

3. **风险类目标**
   - `max_drawdown_5d`
   - `max_drawdown_20d`
   - `volatility_forward_10d`

4. **相对收益类目标**（更推荐）
   - `alpha_5d`：相对行业/指数超额收益
   - `alpha_20d`

### 3.2 第二层：预测要“分场景”
A 股的收益分布高度依赖市场状态，因此不建议做单一通用模型。

建议至少按以下维度分层：
- 市场 regime（牛 / 熊 / 震荡 / 高波动 / 低波动）
- 股票风格（大盘 / 小盘 / 高换手 / 低换手）
- 板块类型（科技 / 周期 / 金融 / 医药 / 资源等）
- 事件状态（连板、涨停打开、热点题材、龙虎榜活跃等）

### 3.3 第三层：预测要“可组合决策”
用户真正需要的通常不是单一概率，而是：
- 胜率
- 盈亏比
- 回撤风险
- 是否值得交易
- 在当前市场环境里应不应该提高仓位

所以最终输出建议不是一个分数，而是一个“决策包”：
- 上涨概率
- 预期收益
- 预期回撤
- 风险收益比
- 信号置信度
- 推荐动作（关注 / 观察 / 等待 / 回避）

## 4. 推荐的高级预测架构（基于现有数据最可落地）

我建议采用 **三层模型架构**，而不是一步冲深度学习。

---

### 4.1 第 1 层：特征工程层（Feature Layer）

目标：把现有原始表，整理成统一训练样本。

#### A. 价格与动量特征
- 日/周/月收益率：1/3/5/10/20/60 日
- 均线偏离率：close/ma5、close/ma20、close/ma60
- 均线斜率
- 新高/新低距离
- 突破/回踩状态
- ATR / 波动率 / 振幅
- RSI / MACD / KDJ / 布林带宽度与位置

#### B. 成交与博弈特征
- 换手率
- 量比
- 量能趋势（5/10/20 日均量比）
- 放量上涨 / 缩量回调 / 天量滞涨 等模式
- 日内尾盘强度（如果分钟数据覆盖足够）

#### C. 估值与基本面特征
- PE/PB/PS 所在历史分位
- 市值分层
- ROE / ROA / 毛利率 / 资产负债率
- 营收、利润、扣非利润同比 / 环比变化
- 经营现金流质量

#### D. 资金与情绪特征
- 主力净流入、连续净流入天数
- 龙虎榜热度
- 涨停/连板/炸板事件
- 行业内资金热度
- 市场情绪阶段标签

#### E. 横截面对比特征
- 相对行业强度
- 相对指数强度
- 同行业排名
- 在全市场的动量 / 波动 / 资金流分位

#### F. 历史推荐行为特征
- 过去是否频繁被推荐为强势票
- 历史被推荐后的平均收益/回撤
- 模型历史置信度稳定性

---

### 4.2 第 2 层：基础预测模型层（Base Models）

这里建议不要只押一个模型，而是做“多模型并行 + 集成”。

#### 模型 A：规则模型（保留）
作用：
- 可解释
- 保底
- 在数据缺失时依旧可用
- 作为 ensemble 的一部分

#### 模型 B：Gradient Boosting 树模型（强烈推荐优先落地）
推荐候选：
- LightGBM
- XGBoost
- CatBoost

原因：
- 对表格型金融数据非常适合
- 能处理非线性与特征交互
- 对缺失值鲁棒
- 训练/推理快
- 可做特征重要性分析
- 在当前数据形态下，通常比深度学习更实用

可分别训练：
- 分类模型：预测 `y_up_5d / y_up_20d`
- 回归模型：预测 `ret_5d / ret_20d`
- 风险模型：预测 `max_drawdown_5d`

#### 模型 C：相似样本模型（保留升级）
当前已有 KNN 风格思路，建议升级为：
- 特征标准化后做 KNN / ANN 相似样本检索
- 返回历史样本未来收益分布，而不仅是单一概率
- 输出：
  - 相似样本数
  - 中位数收益
  - 胜率
  - 分位回撤
  - 市场环境一致性

这个模块特别适合做“解释层”：
- 不是只说“看涨 63%”
- 而是说“过去最像现在的 80 个样本里，未来 5 日有 62% 为正，平均收益 +2.3%，95 分位最大回撤 -4.1%”

#### 模型 D：时序深度学习模型（后置，不建议第一阶段就上）
可选：
- TCN
- LSTM / GRU
- Temporal Fusion Transformer

但建议放到后期，原因：
- 当前项目的主要优势是结构化表格 + 行情 + 资金 + 财务 + 事件；
- 在这种数据形态下，树模型通常先打出更好的性价比；
- 深度学习更适合在样本量、清洗质量、特征统一程度都上来之后再做。

---

### 4.3 第 3 层：集成决策层（Meta / Ensemble Layer）

这是最终让系统“更高级”的关键。

建议输出不要只依赖一个模型，而是融合：
- 规则模型概率
- 树模型概率
- 相似样本概率
- 风险模型输出
- 市场 regime 权重

#### 集成方式建议

##### 方案 1：加权融合（第一阶段最好落地）
按历史验证表现动态赋权：
- `P_final = w1 * rule + w2 * lgbm + w3 * knn`
- 权重可按：
  - 全局历史命中率
  - 当前 regime 历史表现
  - 当前行业历史表现
  - 当前样本置信度

##### 方案 2：二层元模型（第二阶段）
训练一个 meta model，把以下作为输入：
- rule_prob
- gbdt_prob
- knn_prob
- predicted_drawdown
- neighbor_count
- market_regime
- industry
- volatility bucket

输出：
- 最终上涨概率
- 最终交易评分

这样比手工调权更稳。

## 5. 结合现有数据，最推荐的升级路径

为了避免一上来做得过重，我建议按 4 个阶段推进。

### Phase 1：先把“样本体系”做标准化（必须先做）
产出：统一训练样本表 / Parquet 数据集。

建议建立一张或一套离线特征表，例如：
- `feature_stock_daily_ml`

每一行对应：
- 某股票在某交易日的快照特征
- 未来 5/10/20 日标签

字段包括：
- 基础标识：`trade_date, ts_code, industry, market`
- 技术面特征
- 资金面特征
- 财务面特征
- 情绪事件特征
- 标签（未来收益/方向/回撤）

这是后面所有模型的地基。

### Phase 2：先上线 GBDT 模型，不急着上深度学习
建议优先训练：
- `y_up_5d` 二分类
- `y_up_20d` 二分类
- `ret_5d` 回归
- `max_drawdown_5d` 回归或分箱分类

用 walk-forward / expanding window 做时序验证，避免未来函数。

### Phase 3：升级相似样本解释器
把当前 KNN 风格模块升级成“历史情境检索器”：
- 给出最像的历史样本区间
- 给出未来收益分布
- 给出当前置信度
- 给出当前场景属于哪类 regime

### Phase 4：做集成与实盘评估
形成最终线上输出：
- 概率
- 预期收益
- 预期风险
- 置信度
- 动作建议
- 解释文本

## 6. 推荐的评价指标（比“命中率”更重要）

如果只是看“涨跌命中率”，很容易误导。建议至少同时看：

### 6.1 分类指标
- AUC
- PR-AUC
- LogLoss
- Brier Score（概率校准很重要）
- Calibration Curve

### 6.2 收益指标
- TopN 组合未来 5/20 日平均收益
- 多空分层收益
- 信息系数（IC / Rank IC）
- 累计收益
- Sharpe / Sortino

### 6.3 风险指标
- 最大回撤
- 波动率
- 胜率
- 盈亏比
- Tail risk（极端回撤）

### 6.4 稳定性指标
- 分行业表现
- 分市场状态表现
- 分年份表现
- 分市值桶表现

一个真正能上线的模型，不是某段时间特别准，而是：

> **跨年份、跨风格、跨行业都不至于失真太厉害。**

- 未来 5 日预期最大回撤：-3.4%

- 当前状态：震荡偏强 / 趋势延续观察中

### 7.2 决策标签
- ✅ 重点关注
- 👀 观察
- 🟡 等待
- ⚠️ 回避

### 7.3 解释层
- 规则层结论：均线结构、MACD、量比、强弱位置
- 模型层结论：GBDT 概率 / 相似样本胜率
- 风险层结论：回撤风险、波动风险、是否接近压力区
- 历史样本解释：类似过去哪些阶段、未来通常怎么走

## 8. 对“更扎实、更高级”的具体建议

如果目标是“更扎实”，我建议优先级是：

1. **先统一样本与标签定义**
2. **先做时序回测框架**
3. **优先上 GBDT，而不是直接深度学习**
4. **保留规则模型做兜底与解释**
5. **把 KNN 类模块升级成相似样本解释器**
6. **把最终输出从单一分数升级成：概率 + 收益 + 回撤 + 置信度**

如果目标是“更高级”，我建议高级体现在：
- 有 regime 分层
- 有风险预测
- 有概率校准
- 有相似样本解释
- 有组合级验证
- 有模型集成

而不是只追求“模型名字高级”。

- 有历史推荐结果沉淀；
- 有较丰富的行情 / 财务 / 资金 / 事件数据源。

所以后续最优路线不是推倒重来，而是：

> **把现有规则系统升级成一个“有统一样本、有标准标签、有回测验证、有多模型集成、有风险输出”的预测系统。**

这条路既扎实，也足够高级，而且最重要的是：

> **它和你现在已有的数据资产高度匹配，落地成功率最高。**
