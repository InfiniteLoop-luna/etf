# 自选基金独立页面改版设计

日期：2026-06-25

## 目标

将当前嵌在“公募持仓热股”页面底部的自选基金看板，改造成“基金”模块下的独立页面“⭐ 自选基金”。

新页面沿用“⭐ 自选管理”的深色科技看板语言与管理交互，但只展示基金现有基础信息和持仓快照，不在本期新增净值、收益率、回撤等行情数据链路。

## 范围

本期包含：

- 在“基金”侧边栏模块新增独立的 `fund_watchlist` 页面。
- 从“公募持仓热股”页面移除内嵌自选基金看板。
- 使用中文重写自选基金页面的标题、提示、按钮和错误信息。
- 提供总览指标、看板/表格视图、排序、卡片聚焦详情和批量删除。
- 复用当前 `app_user_watchlist` 中 `security_type="fund"` 的用户数据。
- 复用现有基金基础信息与持仓快照查询能力。
- 覆盖未登录、空列表、局部数据失败和移动端布局。

本期不包含：

- 基金净值和实时行情接入。
- 近一月、近一年收益或最大回撤。
- 基金评分、买卖建议或收益预测。
- 新的数据表、后台同步任务或第三方数据源。

## 页面入口与导航

“基金”模块新增：

- 页面 ID：`fund_watchlist`
- 页面名称：`⭐ 自选基金`
- 页面说明：`管理个人自选基金与持仓结构`
- 工具栏密度：`standard`

`src/navigation_config.py` 增加语义明确的基金页面标签常量，建议命名为 `ETF_FUND_WATCHLIST_PAGE_LABEL`。

桌面和移动导航都必须将该标签路由到新的 `render_fund_watchlist_tab()`。原“公募持仓热股”页面不再调用自选基金渲染函数。

## 页面结构

页面按以下顺序组织：

1. 标题与说明
2. 四项组合总览
3. 视图、排序与批量管理工具栏
4. 基金卡片网格或表格视图
5. 当前聚焦基金详情

页面默认使用看板视图，并聚焦当前排序结果中的第一只基金。

### 标题

- 标题：`⭐ 自选基金`
- 说明：`追踪自选基金的持仓结构、披露进度与集中度变化`

### 组合总览

展示四项指标：

- 自选基金：当前用户的基金数量。
- 最新披露期：所有已加载基金中最新的持仓披露期。
- 平均 Top10 集中度：各基金前十大持仓权重之和的平均值。
- 持仓变动：当前快照中“新进/增持”与“减持”的合计数量。

缺失数据使用 `-`，不得用零伪装未知值。

### 工具栏

工具栏包括：

- 视图：`看板` / `表格`
- 排序：`Top10 集中度`、`基金规模`、`持仓市值`、`披露日期`
- 批量管理入口

默认按 Top10 集中度降序排列。排序对卡片和表格使用同一份展示模型。

## 基金卡片

每张卡片展示：

- 基金名称
- 基金代码
- 基金类型
- Top10 集中度
- 基金规模
- 前十大持仓市值
- 新进数量
- 增持数量
- 减持数量
- 最新披露日期

点击卡片将其设为聚焦基金，并更新页面下方详情；不跳转到其他页面。

颜色语义：

- 青色：选中、焦点和中性重点。
- 绿色：增持、较低集中度。
- 红色：减持、较高集中度。

集中度颜色只表达组合结构差异，不表示基金优劣，也不生成投资建议。

## 表格视图

表格视图展示与卡片一致的核心字段，并保持当前排序结果。

建议列：

- 基金名称
- 基金代码
- 基金类型
- 基金规模
- 持仓市值
- Top10 集中度
- 新进
- 增持
- 减持
- 最新披露日期
- 加入日期

表格选中行为应能更新聚焦基金；如果 Streamlit 当前表格能力无法稳定提供行选择，则保留独立的“报告/操作焦点”选择器作为无损降级。

## 聚焦基金详情

详情区域分为左右两部分。

左侧概览展示：

- 基金名称与代码
- 基金管理人
- 基金类型
- 最新披露日期
- Top10 集中度环形指标
- 持仓数量
- 加入自选日期

右侧展示前十大持仓表：

- 股票名称
- 股票代码
- 持仓市值
- 持仓权重
- 持仓变化：新进、增持、减持、稳定

持仓为空时显示基金级空状态，不隐藏整个页面。

## 批量管理

批量管理复用自选股票的交互模式：

1. 点击“进入批量选择”。
2. 卡片或行显示选择控件。
3. 提供“全选”和“取消全选”。
4. 删除按钮显示已选数量。
5. 删除前展示二次确认和基金名称。
6. 删除成功后清理选择状态、刷新自选数据并重新选择有效焦点。

所有删除操作使用 `remove_watchlist_items_batch()`，项目类型固定为 `fund`。

## 数据模型与加载

页面先加载用户自选基金，再为每只基金补充基础信息和最新持仓快照。

数据来源：

- `list_watchlist_items(current_username, security_type="fund")`
- `search_funds(fund_code, ...)`
- `query_fund_preference_snapshot(fund_code=..., top_n=10, ...)`

渲染前统一转换成页面展示模型。建议每只基金的展示模型包含：

- `fund_code`
- `fund_name`
- `fund_type`
- `management`
- `issue_amount`
- `latest_end_date`
- `added_at`
- `holding_count`
- `holding_market_value`
- `top10_ratio`
- `new_count`
- `increase_count`
- `decrease_count`
- `stable_count`
- `holdings`
- `load_error`

页面总览、卡片、表格和详情都从该模型读取，避免在多个渲染分支重复查询和计算。

同一次 Streamlit 会话中，切换焦点、排序或视图时应复用已加载模型。缓存键至少包含用户名和当前基金代码集合；删除基金后必须使缓存失效。

## 组件边界

保持渲染职责清晰，避免继续扩大单一函数：

- `load_fund_watchlist_dashboard_data(...)`：加载并构建展示模型。
- `build_fund_watchlist_summary(...)`：计算组合总览。
- `render_fund_watchlist_tab()`：页面编排与状态协调。
- `render_fund_watchlist_dashboard(...)`：总览、工具栏和卡片/表格区域。
- `render_fund_watchlist_focus_detail(...)`：聚焦基金详情。
- `render_fund_watchlist_batch_actions(...)`：批量选择与删除确认。

具体函数拆分可根据现有 `app.py` 组织略作调整，但数据加载、页面编排、详情渲染和删除操作不得重新混为一个大循环。

## 视觉系统

复用现有自选股票看板的设计语言：

- 深蓝黑背景。
- 蓝色描边和轻微内发光。
- 青色焦点色。
- 绿色和红色状态色。
- 紧凑但可读的指标卡与基金卡片。
- 同一套按钮、选中态、批量模式和移动端折行规则。

基金页面使用独立 CSS 作用域，避免影响自选股票和其他 Streamlit 页面。可以共享设计令牌，但不复用包含股票专属尺寸或字段假设的选择器。

## 状态与异常处理

- 未登录：提示“请先登录用户名，再查看和管理你的自选基金。”
- 空列表：提示从“公募持仓热股”的基金持仓查询区域添加基金。
- 自选列表加载失败：显示页面级错误并停止后续加载。
- 基金数据库连接失败：显示页面级错误并停止详情加载。
- 单只基金基础信息或持仓加载失败：记录在该基金的 `load_error`，保留其他基金。
- 单只基金部分字段缺失：显示 `-`，不抛出整个页面异常。
- 当前焦点在删除或筛选后不存在：自动选择剩余结果的第一只基金。
- 当前筛选结果为空：显示筛选级空状态，不清空用户自选。

## 响应式设计

桌面端：

- 总览四列。
- 基金卡片三列，宽屏可根据容器宽度增加列数。
- 聚焦详情使用左侧概览、右侧持仓表的双栏布局。

窄屏和移动端：

- 总览改为两列。
- 基金卡片改为单列或两列。
- 聚焦详情改为上下布局。
- 工具栏允许换行。
- 表格允许横向滚动，不压缩到不可读。
- 批量删除确认按钮保持可点击宽度。

## 测试与验收

自动化测试至少覆盖：

- 侧边栏存在 `fund_watchlist`，并归属“基金”模块。
- 导航配置暴露新的基金自选页面标签。
- 桌面和移动路由都调用 `render_fund_watchlist_tab()`。
- “公募持仓热股”不再调用自选基金看板。
- 页面只加载 `security_type="fund"`。
- 未登录和空列表状态。
- 总览计算：数量、最新披露期、平均集中度、持仓变动。
- 展示模型对缺失字段和单基金加载失败的处理。
- 排序字段和默认顺序。
- 卡片焦点切换。
- 看板/表格视图切换。
- 批量删除始终提交 `(fund_code, "fund")`。
- 删除后焦点与缓存状态恢复。
- 页面关键中文文案和基金字段存在。

浏览器验收至少覆盖：

- 桌面宽度下第一屏层级、卡片密度和详情布局。
- 移动宽度下无横向页面溢出。
- 点击多个基金卡片能稳定切换详情。
- 视图切换和四种排序可用。
- 批量选择、全选、取消和二次确认删除可用。
- 单只基金无持仓或加载失败时，其他基金仍正常展示。
- 视觉风格与自选股票看板一致，但字段全部为基金语义。

## 成功标准

- 用户可从“基金”模块直接进入独立自选基金页面。
- 页面不再依附“公募持仓热股”。
- 用户能在一个页面完成基金组合概览、比较、聚焦查看和批量删除。
- 页面仅使用现有基金基础信息和持仓快照即可完整工作。
- 异常或缺失数据不会导致整页不可用。
- 桌面和移动端均保持深色科技看板的可读性与操作完整性。
