# 指数监测设计文档

日期：2026-05-06  
状态：已在对话中完成用户确认  
范围：基于 `新增证券分析表单-20260505.xlsx` 的 `股票指数` sheet，在 Web 页面 `宏观` 分类下新增 `📊 指数监测` 仪表盘，并支持手工录入与 Excel 导入。

## 1. 背景

当前应用在 `宏观` 分类下已经有：

- `🌏 宏观经济`
- `🏦 本外币存款`

用户希望继续把参考工作簿 `新增证券分析表单-20260505.xlsx` 中的 `股票指数` 数据做成一个可长期维护、可视化展示、可在网页中录入和导入的仪表盘页面，并挂载到 `宏观` 分类下。

该 sheet 主要围绕多个指数（月度粒度）提供三组数据：

1. `当月情况`
2. `环比变动情况`
3. `同比变动情况`

已识别出的核心指标包括：

- 指数名称
- 涨幅
- 开盘价格
- 收盘价格
- 最低点
- 最高点
- 期末静态市盈率
- 期末动态市盈率
- 静态市盈率变化率
- 动态市盈率变化率

## 2. 用户已确认的设计方向

对话中用户已明确选择：

- 页面位置放在 `宏观` 下面，而不是 `ETF`
- 页面名称使用 `📊 指数监测`
- 首页采用 `总览型` 仪表盘
- 手工维护支持两种方式：
  - 按月份批量录入
  - 按单个指数补录 / 修改
- 顶部 KPI 采用 `混合总览`
- 时间序列展示采用 `双主图`
  - 价格趋势
  - 估值趋势

## 3. 目标

- 在 `宏观` 分类下新增 `📊 指数监测` 页面
- 将 `股票指数` sheet 转为适合 Web 展示和维护的结构化月度数据
- 提供适合日常浏览的总览型仪表盘
- 支持未来持续录入新月份数据
- 支持 Excel 批量导入并在写入前预览
- 支持对单个月份或单个指数进行后续修订

## 4. 非目标

首版不做以下内容：

- 外部行情 API 自动同步
- 自动生成研判结论或文字点评
- 多工作簿、多版本对账能力
- 复杂因子分析、热力地图谱、回测模块

## 5. 页面挂载与信息架构

### 5.1 导航位置

在 `宏观` 二级页面中新增：

- `🌏 宏观经济`
- `🏦 本外币存款`
- `📊 指数监测`

桌面与 iPhone 模式都要保持一致的导航结构。

### 5.2 页面整体结构

`📊 指数监测` 首页采用“总览 + 维护”的结构，推荐分为五个区域：

1. 顶部操作与筛选区
2. KPI 总览区
3. 双主图区
4. 横向对比表区
5. 底部数据维护区

## 6. 首页布局设计

### 6.1 顶部操作与筛选区

建议提供以下控件：

- 月份选择
- 指数多选
- 时间范围：
  - 最近12个月
  - 最近24个月
  - 全部
- `新增月份`
- `单指数补录/修改`
- `批量导入 Excel`

设计原则：

- 默认先服务“看数据”，录入动作可见但不抢占首屏主体
- 页面首屏优先让用户读出“这个月市场和估值大概怎么样”

### 6.2 KPI 总览区

首页顶部展示混合型 KPI，建议为：

- 最新月份
- 指数平均涨幅
- 当月最强指数
- 当月最弱指数
- 平均静态 PE
- 平均动态 PE

补充规则：

- “最强 / 最弱指数”默认按 `当月涨幅` 判断
- 如果当月缺失某项数据，则显示 `-`
- 平均估值只基于当月存在有效值的指数计算

### 6.3 双主图区

首页中部展示两张主图：

#### 图 1：价格趋势

目标：回答“指数价格最近怎么走”

默认展示：

- 收盘价趋势

可扩展字段：

- 开盘价
- 最低点
- 最高点

默认行为：

- 支持多指数对比
- 默认按所选指数绘线
- 默认使用当前时间范围筛选

#### 图 2：估值趋势

目标：回答“估值水平最近怎么走”

默认展示：

- 期末静态市盈率
- 期末动态市盈率

设计原则：

- 价格趋势与估值趋势拆成两张图，不混在一张图里
- 首页强调“价格”和“估值”两条主线并行阅读

### 6.4 横向对比表区

首页下方提供一张指数横向对比表，一行一个指数。

建议默认列包含：

- 指数名称
- 当月涨幅
- 开盘价格
- 收盘价格
- 最低点
- 最高点
- 环比涨幅变化
- 同比涨幅变化
- 静态 PE
- 动态 PE
- 静态 PE 变化率
- 动态 PE 变化率

设计目标：

- 适合快速比较上证指数、深证成指、沪深300等指数的横向差异
- 既能看市场表现，也能看估值变化

### 6.5 底部维护区

首页底部放数据维护能力，包含：

- 按月份批量录入
- 按单个指数补录 / 修改
- Excel 导入预览
- 历史明细表

这样可以把“运营维护”与“仪表盘阅读”分层处理，不破坏首页的阅读节奏。

## 7. 数据模型设计

### 7.1 建模原则

不要保留 Excel 当前的横向分组布局作为系统主存储结构。  
应将其标准化为“每个月份、每个指数一行”的明细表。

即：

- 一条记录 = `month + index_name`

### 7.2 推荐逻辑字段

#### 主键与标识字段

- `month`
- `index_name`

#### 当月情况

- `monthly_change_pct`
- `open_price`
- `close_price`
- `low_price`
- `high_price`
- `static_pe`
- `dynamic_pe`

#### 环比变动情况

- `mom_change_pct`
- `mom_open_price`
- `mom_close_price`
- `mom_low_price`
- `mom_high_price`
- `mom_static_pe`
- `mom_dynamic_pe`
- `mom_static_pe_change_rate`
- `mom_dynamic_pe_change_rate`

#### 同比变动情况

- `yoy_change_pct`
- `yoy_open_price`
- `yoy_close_price`
- `yoy_low_price`
- `yoy_high_price`
- `yoy_static_pe`
- `yoy_dynamic_pe`

#### 审计字段

- `source_type`
- `source_file`
- `created_at`
- `updated_at`

### 7.3 存储策略

与 `本外币存款` 页一致，首版直接写入 PostgreSQL。

原因：

- 该项目已经有 PostgreSQL 和 SQLAlchemy 的既有模式
- 适合做按月份 upsert、按指数筛选、历史查询
- 更便于手工补录和导入预览

继续沿用现有项目的连接配置约定：

- `ETF_PG_HOST`
- `ETF_PG_PORT`
- `ETF_PG_DATABASE`
- `ETF_PG_USER`
- `ETF_PG_PASSWORD`
- `ETF_PG_SSLMODE`

不重复发明新的数据库配置方式。

### 7.4 推荐表

建议使用独立表，例如：

- `macro_index_monitor_monthly`

推荐唯一键：

- `(month, index_name)`

这样支持：

- 同一月份多个指数并存
- 同一指数后续补录覆盖
- Excel 导入按主键 upsert

## 8. Excel 导入设计

### 8.1 导入来源

首版支持导入与 `新增证券分析表单-20260505.xlsx` 相同结构的 `股票指数` sheet。

### 8.2 解析原则

该 sheet 是“横向分组表头”，解析器需要识别：

- 指数名称列
- 当月情况区块
- 环比变动情况区块
- 同比变动情况区块

解析后输出标准化结果：

- 一行一个 `month + index_name`

### 8.3 导入预览

写入前必须给用户预览，区分：

- 新增记录
- 覆盖已有记录
- 解析失败记录

失败记录必须说明原因，例如：

- 缺失月份
- 缺失指数名称
- 表头无法识别
- 数值列异常

### 8.4 冲突处理

当导入数据的 `(month, index_name)` 已存在时：

- 支持 `跳过`
- 支持 `覆盖`

最终写库采用显式确认后的 upsert。

## 9. 手工录入设计

用户要求支持两种维护方式，首版按此落地。

### 9.1 方式一：按月份批量录入

适用于：

- 新增一个完整月份
- 一次性录入该月份多个指数

交互建议：

1. 点击 `新增月份`
2. 选择月份
3. 展示一个可编辑表格或分组表单
4. 一次性录入多个指数
5. 保存后整体写入

### 9.2 方式二：按单个指数补录 / 修改

适用于：

- 后续修订某一个指数的某个月数据
- 对导入后的小范围错误做修正

交互建议：

1. 选择月份
2. 选择指数
3. 打开单指数编辑表单
4. 修改后保存

## 10. 展示口径与计算规则

### 10.1 首页指标口径

- 指数平均涨幅：基于当月所有有值指数的 `monthly_change_pct`
- 最强指数：当月涨幅最大
- 最弱指数：当月涨幅最小
- 平均静态 PE：基于有值指数平均
- 平均动态 PE：基于有值指数平均

### 10.2 趋势图口径

价格趋势图：

- 默认取 `close_price`
- 多指数对比时，一条线对应一个指数

估值趋势图：

- 每个指数可有两条线：
  - `static_pe`
  - `dynamic_pe`

若后续视觉过于拥挤，可在实现阶段增加“只看一个估值指标”的切换，但这不影响当前设计。

### 10.3 不反推 Excel 现成环比 / 同比字段

对于 `股票指数` 页，Excel 已经明确提供了环比和同比字段。  
首版设计中，这些字段应直接存储并直接展示，不在页面端重新反推。

原因：

- 与业务原表口径保持一致
- 减少口径误差
- 方便未来人工维护

## 11. 空状态与边界状态

### 11.1 无数据

页面显示空状态，并给出操作入口：

- `新增月份`
- `批量导入 Excel`

### 11.2 只有一个月份

- KPI 正常显示
- 趋势图可只显示一个点
- 历史图不报错

### 11.3 某些指数字段缺失

- 表格中显示 `-`
- 图表跳过空值
- 不阻断整页展示

### 11.4 导入部分失败

- 成功记录可以预览
- 失败记录单列列出原因
- 不允许静默吞掉失败数据

## 12. 技术实现建议

建议沿用 `本外币存款` 页的整体实现模式，新增以下单元：

- 一个新的指数监测数据存储模块
- 一个新的 `股票指数` sheet 解析器
- 一组测试文件
- `宏观` 导航接入
- 一个新的页面渲染函数

推荐职责划分：

- `store`：数据库连接、建表、读取、upsert、聚合
- `importer`：专门解析 `股票指数` sheet
- `app.py`：渲染页面、交互、图表、录入流程
- `tests`：解析、聚合、展示口径、冲突处理

## 13. 测试策略

### 13.1 解析测试

- 正确识别 `股票指数` sheet
- 正确识别月份
- 正确识别指数名称
- 正确把三组区块摊平成标准字段
- 缺表 / 异常表头时给出明确错误

### 13.2 数据逻辑测试

- KPI 统计正确
- 最强 / 最弱指数判定正确
- 导入记录按 `(month, index_name)` 正确分类为新增或覆盖

### 13.3 UI 行为测试

- `宏观` 下出现 `📊 指数监测`
- 空状态可见
- 明细表默认按指数展示
- 导入预览与写入流程可走通
- 单指数编辑可覆盖原记录

## 14. 首版交付范围

首版建议交付：

- `宏观 / 📊 指数监测` 页面
- 顶部筛选与操作区
- KPI 总览
- 价格趋势图
- 估值趋势图
- 横向对比表
- 手工录入（两种模式）
- Excel 导入预览与覆盖写入
- PostgreSQL 存储

首版延后项：

- 自动同步
- 分层权限
- 高级分析与自动结论

## 15. 决策总结

本次确认后的设计方案为：

- 页面名称：`📊 指数监测`
- 页面位置：`宏观` 二级页
- 页面类型：总览型仪表盘
- 手工维护：支持按月份批量录入与单指数补录 / 修改
- 首页 KPI：混合总览
- 主图结构：价格趋势 + 估值趋势双主图
- 导入方式：解析 `股票指数` sheet 后标准化入库
- 存储方式：PostgreSQL，主键为 `(month, index_name)`
